## 2. K-means 算法原理

说明 K-means 的目标函数、基本步骤和收敛条件。

重点内容:

1. SSE/inertia 目标函数。
2. 聚类中心初始化。
3. 样本分配阶段。
4. 聚类中心更新阶段。
5. 迭代终止条件: 达到 `max_iter` 或中心移动距离小于 `tol`。

可使用公式:

```text
SSE = sum_i ||x_i - c_label(i)||^2
```

## 3. 串行算法复杂度分析

设样本数量为 `n`，特征维度为 `dim`，聚类数量为 `k`，实际迭代次数为 `I`。

核心分析:

1. 每轮迭代需要计算 `n` 个样本到 `k` 个中心的距离。
2. 每次距离计算需要遍历 `dim` 个特征。
3. 单轮主要时间复杂度为 `O(n * k * dim)`。
4. 总时间复杂度为:

```text
O(I * n * k * dim)
```

空间复杂度可分析数据集、标签数组、聚类中心和临时统计数组。

## 4. 并行算法设计

说明并行化对象和数据竞争处理方式。

重点内容:

1. 样本分配阶段具有天然并行性。
2. 每个线程处理一部分样本。
3. 每个线程维护自己的 `local_sums` 和 `local_counts`。
4. 并行阶段结束后合并局部统计结果。
5. 根据合并结果更新全局聚类中心。

可强调本设计避免了在样本循环中使用锁或临界区，从而降低同步开销。

## 5. OpenMP 实现方式

说明代码中使用的 OpenMP API 和指令。

可写内容:

1. 使用 `omp_set_num_threads(threads)` 设置线程数。
2. 使用 `#pragma omp parallel` 创建并行区域。
3. 使用 `#pragma omp for schedule(static)` 分配样本循环。
4. 使用线程编号 `omp_get_thread_num()` 访问对应的局部统计数组。
5. 使用 `reduction(+:inertia)` 并行计算最终 SSE。

可结合 `src/kmeans_openmp.cpp` 展示关键代码片段。

## 9. 加速比分析

加速比公式:

```text
Speedup = T_serial / T_parallel_p
```

分析角度:

1. 对每个 `n`，以同一规模下串行时间作为基准。
2. 比较 `p = 1, 2, 4, 8` 时的加速比变化。
3. 解释实际加速比通常小于理论线程数的原因。
4. 结合并行开销、不可并行部分和内存带宽进行分析。

## 10. 并行效率分析

并行效率公式:

```text
Efficiency = Speedup / p
```

分析角度:

1. 并行效率反映平均每个线程的利用程度。
2. 线程数增加时，效率通常会下降。
3. 数据规模较小时，并行效率容易偏低。
4. 数据规模较大时，计算量增加，可以摊薄同步和合并开销，效率可能更高。
